# 🏛️ C4 Model - Arquitetura do Sistema

O modelo C4 descreve a arquitetura em níveis de abstração, padronizando a comunicação técnica.

## Nível 1: Contexto de Sistema
Descreve as interações macro e integrações externas da aplicação.

```mermaid
C4Context
  title Nível 1: Diagrama de Contexto de Sistema
  
  Person(cs, "Customer Success", "Usuário da aplicação")
  System(app, "CS Manager App", "Sistema de Gestão e Comissionamento")
  System_Ext(eproc, "Tribunal Eproc", "Sistema Judicial Oficial")
  System_Ext(gemini, "Google Gemini", "LLM Inteligência Artificial Multimodal")

  Rel(cs, app, "Lança comissões e visualiza dashboard", "HTTPS")
  Rel(app, eproc, "Automação Headless para captura de status", "HTTPS/Selenium")
  Rel(app, gemini, "Processa áudios e vídeos via API", "REST/JSON")
```

## Nível 2: Container (MVC Expandido)
Mostra como o software é dividido internamente, detalhando os módulos.

```mermaid
C4Container
  title Nível 2: Diagrama de Containers
  
  Person(cs, "Customer Success", "Acessa via Desktop (Navegador)")
  
  System_Boundary(c1, "CS Manager App Local") {
    Container(spa, "Frontend Application", "React, Vite, CSS", "Interface Glassmorphism de Lançamentos (View)")
    Container(api, "API Application", "Python, FastAPI", "Regras de negócio, Comissionamento e Roteamento (Controller)")
    ContainerDb(db, "Database", "SQLite", "Armazena dados de forma embutida e local (Model)")
    Container(scraper, "Scraper Worker", "Python, Undetected Chromedriver", "Worker paralelo para navegação do Eproc")
  }

  System_Ext(gemini, "API Gemini", "Processamento de Áudio/Vídeo")

  Rel(cs, spa, "Interage", "Navegador Web")
  Rel(spa, api, "Faz chamadas de API", "JSON/HTTPS")
  Rel(api, db, "Lê e Escreve Lançamentos", "SQL")
  Rel(api, scraper, "Despacha tarefas e planilhas", "Subprocess")
  Rel(api, gemini, "Transcreve e Sumariza", "API HTTP/2")
```
