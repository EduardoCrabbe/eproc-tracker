# 🗄️ Entidade-Relacionamento (ERD)

Este é o diagrama lógico relacional do nosso banco de dados embutido (SQLite). 

> [!NOTE]
> A segregação clara da tabela `CUSTOMERS` garante que IDs sensíveis ou CPFs não sejam processados indevidamente nos logs gerais de atendimentos.

```mermaid
erDiagram
    USERS ||--o{ ATTENDANCES : "realiza lançamento"
    CUSTOMERS ||--o{ ATTENDANCES : "recebe atendimento"

    USERS {
        int id PK
        string email "Apenas domínios @reisrevisional.com.br"
        string password_hash
        string role "Gerente, Supervisor, CS"
        int level_cs "Nível de Comissão (1 a 5)"
    }

    CUSTOMERS {
        string id_datajuri PK
        string first_name "Apenas Primeiro Nome (Anônimo)"
    }

    ATTENDANCES {
        int id PK
        int user_id FK
        string customer_id FK
        datetime timestamp
        string type "Quitacao, Video, Audio..."
        string evidence_file "Path local do anexo"
        string ai_summary "Resumo redigido em 1ª Pessoa"
        float commission_value "Calculado via Backend (Nível CS x Tipo)"
    }
```
