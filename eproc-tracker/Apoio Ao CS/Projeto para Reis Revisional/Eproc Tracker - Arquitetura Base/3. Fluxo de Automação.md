# 3. Fluxo de Automação do Eproc Tracker

Abaixo estão os diagramas estruturais e lógicos que mapeiam o ciclo de vida da execução da nossa aplicação, desde o pre-flight check de sanidade até a triagem dos resultados extraídos do tribunal.

> [!NOTE]
> Os diagramas servem como o mapa definitivo da aplicação. Qualquer nova feature de navegação deve ser encaixada logicamente neste fluxo antes da implementação.

## 🔄 Diagrama Geral das Etapas (Lifecycle)

Este fluxograma explica passo a passo a tomada de decisão do robô.

```mermaid
flowchart TD
    %% Nós principais
    A([Início: Ativação do Robô]) --> B{Pre-Flight Check<br>qa_tester.py}
    
    %% Fluxo de Falha Pre-Flight
    B -->|Contrato Quebrado| C[Erro: Planilha Inválida / Portal Offline]
    C --> Z([Encerramento Antecipado])
    
    %% Fluxo Feliz Pre-Flight
    B -->|Check OK| D[(Leitura da Planilha<br>base_clientes.xlsx)]
    D --> E[Inicia Motor Scraper<br>via Context Manager]
    
    %% Bypass Anti-Bot
    E --> F{Bypass Anti-Bot<br>Undetected Chromedriver}
    
    F -->|Necessária Intervenção| G[Espera Resolução do CAPTCHA<br>input estático no terminal]
    G --> H[Fluxo de Navegação Liberado]
    
    F -->|Bypass Limpo| H
    
    %% Consultas
    H --> I{Seletor de Busca}
    I -->|Número de Processo Disponível| J[Busca via NU]
    I -->|Fallback Opcional| K[Busca via CPF]
    
    J --> L[Injeta Dados e Clica Consultar]
    K --> L
    
    %% Raspagem
    L --> M[Aguardando Renderização do Tribunal...]
    M --> N[Extração do HTML Bruto<br>Ação e Movimentações]
    
    %% Triagem
    N --> O{Regras de Triagem}
    O -->|Busca e Apreensão /<br>Mandado Crítico| P[🚨 Alerta Vermelho / Destaque]
    O -->|Outras Movimentações| Q[✅ Registro Normal]
    
    %% Finalização
    P --> R[(Gravar Status Atualizado na Planilha)]
    Q --> R
    
    R --> S[Chama __exit__ e limpa a Memória RAM]
    S --> Fim([Fim da Tarefa])
```

## 🏗️ Diagrama de Responsabilidades (UML de Classes)

O modelo Orientado a Objetos (OOP) separa a responsabilidade de testes da responsabilidade de navegação, mantendo o código altamente coeso e pouco acoplado.

```mermaid
classDiagram
    %% Relação entre classes
    QATester --> EprocScraper : Aciona caso válido
    
    class EprocScraper {
        +bool headless
        +WebDriver driver
        +__enter__() self
        +__exit__(exc_type, exc_val, exc_tb)
        +consultar_processo(numero_processo, cpf) dict
        +aplicar_triagem(acao, movimentacao) string
    }
    
    class QATester {
        +validar_contrato_planilha(caminho) bool
        +testar_conexao_eproc() bool
    }
```
