# 1. Setup Inicial e Preparação do Ambiente

Este documento descreve os primeiros passos e as diretrizes fundamentais para a configuração do ambiente seguro do projeto **Eproc Tracker**. A segurança e o isolamento do ambiente são as principais prioridades nesta fase.

> [!IMPORTANT]
> A correta configuração do Setup Inicial garante que dados sensíveis não sejam expostos e que o script tenha uma execução limpa e previsível em qualquer máquina.

## 🛡️ Git e Versionamento Seguro
A configuração do arquivo `.gitignore` é a nossa primeira linha de defesa. O objetivo principal é blindar o repositório local e remoto contra vazamento de dados de clientes e variáveis do sistema:

- **Arquivos de Cache:** Bloqueio estrito de compilados Python (`__pycache__/`, `*.pyc`).
- **Dados Sensíveis:** Exclusão absoluta de planilhas locais de clientes (`*.xlsx`, `*.csv`) que contenham CPFs ou chaves de processo.
- **Credenciais:** Proteção de variáveis de ambiente e credenciais de banco (`.env`, `secrets.json`).

## 📦 Isolamento de Dependências (Virtual Environment)
Para impedir conflitos de versões e garantir que as atualizações do Eproc Tracker não afetem o ambiente Python global da máquina hospedeira, exigimos o uso de um ambiente virtual isolado (`venv`).

> [!TIP]
> Utilize `python -m venv venv` para criar o ambiente e instale as bibliotecas exclusivamente nele.

**Dependências mapeadas no `requirements.txt`:**
- `selenium`: Motor base para simulação da navegação humana e interação com o portal.
- `webdriver-manager`: Gerenciador dinâmico de binários do navegador, assegurando que o driver corresponda sempre à versão do Chrome instalada.
- `undetected-chromedriver`: Ferramenta essencial de patching anti-bot para evasão de sistemas WAF (como Cloudflare) do Tribunal.
- `openpyxl`: Biblioteca dedicada para leitura e escrita otimizada em memória de planilhas Excel, vital para a leitura da `base_clientes.xlsx`.

## 🚥 Validador Fail-Fast (Pre-Flight Check)
Para economizar recursos de hardware e tempo, construímos o script pre-flight `qa_tester.py`. Ele atua como uma barreira inicial:
1. **Contratos de Dados:** Checa se a planilha de entrada existe e valida se todas as colunas obrigatórias estão presentes.
2. **Health Check:** Realiza um *ping* ou requisição HTTP leve na página do TJSP/Eproc para validar se o portal está respondendo, antes de acionar o motor pesado de automação.

Se alguma destas verificações falhar, o sistema interrompe a execução imediatamente (Fail-Fast), alertando o operador.
